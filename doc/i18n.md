# Adding a New Language

All translations live in a single `translations` object inside
`javascript/html5/src/js/hmi.js`.  Each top-level key of that object is a
BCP 47 language tag (e.g. `en`, `de`, `fr`).  A missing key in a locale
automatically falls back to the English value, so you only need to supply
the keys you actually want to translate.

## Step 1 — Add a translation block in `hmi.js`

Open `javascript/html5/src/js/hmi.js` and locate the `translations` object
near the top of the file.  Add a new entry for your language code immediately
before the closing `};` of that object.  Model it on one of the existing
locales (e.g. `fr` or `it`).

```js
const translations = {
  en: { /* … */ },
  de: { /* … */ },
  // ↓ add your locale here
  nl: {
    tagline: "Ontcijfer de geheime code voordat de pogingen op zijn.",
    // … all other keys …
  },
};
```

### Function keys

Two keys must be functions rather than plain strings.

**`statusBody(board)`** — returns a descriptive sentence depending on the
current game state.  The `board` argument exposes:

- `board.status` — `"won"`, `"lost"`, or `"playing"`
- `board.canSubmit` — `true` when the current guess is full
- `board.settings.codeLength` — configured secret length (4 or 5)
- `board.currentGuess.length` — number of symbols entered so far

```js
statusBody: (board) => {
  if (board.status === "won")
    return "Jij vond het geheim. Start een nieuw spel voor een beter resultaat.";
  if (board.status === "lost")
    return "De laatste poging is verbruikt. Bekijk het onthulde geheim en probeer opnieuw.";
  if (board.canSubmit)
    return "Je raden is compleet. Dien het in om feedback te krijgen.";
  const left = board.settings.codeLength - board.currentGuess.length;
  return `Kies nog ${left} symbool${left === 1 ? "" : "en"}.`;
},
```

**`highscoreFormat(entry)`** — formats a highscore entry for display.
The `entry` argument has `entry.attempts` (integer) and `entry.seconds`
(integer).  Use the imported `formatDuration` helper to convert seconds to a
`mm:ss` string:

```js
highscoreFormat: (entry) =>
  `${entry.attempts} pogingen / ${formatDuration(entry.seconds)}`,
```

## Step 2 — Add a radio button in `index.html`

Open `javascript/html5/src/index.html` and find the language `<fieldset>`:

```html
<fieldset>
  <legend data-i18n="optionsLanguage">Language</legend>
  <label><input type="radio" name="language" value="en" checked />English</label>
  <label><input type="radio" name="language" value="de" />Deutsch</label>
  …
</fieldset>
```

Add a new `<label>` for your language.  The `value` attribute must match the
key you used in the `translations` object.  Write the language name in the
language itself (not in English):

```html
<label><input type="radio" name="language" value="nl" />Nederlands</label>
```

## Step 3 — Verify

Run the app in a browser, open Options, select your new language, and check
every screen (game, rules, options, about) for missing or placeholder text.
Any key you omitted will silently fall back to the English string, so there
are no runtime errors to worry about — just visual gaps to fill in.
